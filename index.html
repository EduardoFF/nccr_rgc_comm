Introduction

The purpose of this document is to describe the framework that is used to build and maintain a multi-hop wireless network infrastructure for the NCCR Grand Challenge on Rescue Robotics.

Proposed hardware

  • ODROID U3 or C1(+) with Ubuntu
  • XBee-PRO 868
  • TP-LINK TL-WN722N
  • Camera, GPS Dongle (optional)

Installation on ODROID using docker

  • Install docker:

    sudo apt-get install lxc aufs-tools cgroup-lite apparmor docker.io

  • check that docker runs (you must have internet connection):

    $ sudo docker run armv7/armhf-debian /bin/echo 'Hello world'

  • build the container:

    $ sudo docker build --no-cache -t nccr_manet https://github.com/EduardoFF/nccr_rgc_manet.git

Installation on PC using docker

  • Install docker according to the instructions for your linux distribution

  • check that docker runs (you must have internet connection):

    $ sudo docker run debian:latest /bin/echo 'Hello world'

  • build the container:

    $ sudo docker build --no-cache -t nccr_manet https://github.com/EduardoFF/nccr_rgc_manet.git#:x86_64

Testing:

On ODROID,

  • make directory for logs, e.g. ~/manet_logs

  • start the manet with the container:

    $ sudo docker run --rm -it --privileged --net="host" -v ~/manet_logs:/logs nccr_manet setup_manet -i wlan0 -n 2 -r /root/default.routes

replace wlan0 by the network interface corresponding to the wireless dongle.

-n 2 assigns the node id.

On PC,

  • make directory for logs, e.g., ~/manet_logs

  • start the manet with the container:

    $ sudo docker run --rm -it --privileged --net="host" -v ~/manet_logs:/logs nccr_manet setup_manet -i wlan0 -n 1 -r /root/default.routes
