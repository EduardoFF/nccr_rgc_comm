Introduction

The purpose of this document is to describe the framework that is used to build and maintain a multi-hop wireless network infrastructure for the NCCR Grand Challenge on Rescue Robotics.

Proposed hardware

  • ODROID U3 or C1(+) with Ubuntu
  • XBee-PRO 868
  • TP-LINK TL-WN722N
  • Camera, GPS Dongle (optional)

Installation using docker

  • Install docker: sudo apt-get install lxc aufs-tools cgroup-lite apparmor docker.io
  • check that docker runs: sudo docker info Containers: 0 Images: 0 Storage Driver: aufs Root Dir: /var/lib/docker/aufs Backing Filesystem: extfs Dirs: 0 Dirperm1 Supported: false Execution Driver: native-0.2 Kernel Version: 3.8.13.28 Operating System: Ubuntu 14.04.2 LTS CPUs: 4 Total Memory: 1.976 GiB Name: my-odroid-name ID: G4RD:VWPC:RNEZ:LGE3:44UZ:S7OF:AKFI:QBY6:FS4N:CHS2:PETD:NX4O

Installing dependencies

LCM:

  • https://github.com/lcm-proj/lcm/releases/download/v1.3.1/lcm-1.3.1.zip
  • unzip lcm-1.3.1.zip

Click:

  • git clone https://github.com/EduardoFF/click.git
  • cd click
  • ./configure --disable-linuxmodule
  • make -C ./userlevel MINDRIVER=RNP_BRIDGE_PKG
  • make -C ./userlevel MINDRIVER=RNP_PKG

libxbee

  • git clone https://github.com/attie/libxbee3.git
  • follow the indicated installation procedure

libgps-dev

  • apt-get install libgps-dev

Glog

  • apt-get install libgoogle-glog-dev

XBee bridge:

  • git clone https://github.com/EduardoFF/rnp_xbee_bridge.git
  • cd rnp_xbee_bridge
  • mkdir build
  • cd build
  • cmake ../src
  • make end_node

Setup the adhoc network

  • connect the wifi dongle and identify the interface name (e.g., wlan1)
  • setup the network interfaces in adhoc mode. Use the setup_adhoc.sh script in click/scripts
  • choose an IP address (e.g., 10.42.43.10/24), channel (3), transmit power (20dBm)
  • run the script: sudo bash setup_adhoc.sh wlan1 10.42.43.10/24 3 20
  • repeat the procedure for the other node, but choose a different IP address
